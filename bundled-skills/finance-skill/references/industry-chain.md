# 产业链映射与卡点（industry-chain）

> **何时参考**：用户问"X 产业链有哪些股票 / 上下游怎么分 / 这条产业链卡点在哪 / 谁是核心环节 / 哪一段最受益"。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

**找到产业链中"价值正在传导但市场未充分定价的环节"**——不允许"全链条受益"的泛化结论。

## 分析步骤

1. **主题驱动来源**：需求驱动 / 技术演进 / 政策支持 / 成本下降 / 海外趋势映射——不同驱动对应不同持久力。
2. **产业链层级拆解**：上游资源 / 原材料 → 中游制造 / 集成 / 平台 → 下游应用 / 客户。明确"谁给谁供给、谁对谁定价、谁最容易受影响"。
3. **核心价值环节识别**（卡点）：
   - 最受益环节
   - 利润弹性最大环节
   - 最容易形成龙头溢价的环节
   - 仅概念映射、业绩贡献有限的环节
4. **标的分层**：
   - 核心龙头：逻辑最直接、收入直接受益
   - 容量中军：稳定增长、机构参与
   - 高弹性小票：市占率 / 产能提升潜力
   - 观察性映射标的：市场愿意交易但受益弱
5. **受益路径具体化**：订单增加 / ASP 提升 / 市占率提升 / 产能利用率改善 / 估值重估——尽量数字化。
6. **风险与伪受益识别**：哪些公司名义相关但实际受益弱、哪些环节虽热门但兑现周期长、哪些标的业绩与主题关联度不高。
7. **优先研究顺序**：先看哪一段、再看哪一段、什么时候应该全链铺开——通常**卡点先动，扩散后到下游 / 跟风**。

## 建议骨架（不强制）

- 产业链全景（层级 + 作用 + 价值分布）
- 核心价值环节与稀缺性
- 受益标的分层
- 投资映射逻辑与受益路径
- 风险与伪受益识别
- 优先研究顺序与验证指标

## 避坑

- 最忌"全链条受益"的泛化结论——找最受益的一段
- 上游价格波动通常快速向下游传导，但下游利润压缩快于恢复
- 产业链中的龙头往往集中在某一环，不是每一环都有龙头
- 新兴主题的产业链常存在"产能过剩"与"供不应求"激烈分歧——谨慎评估
- 国产替代主题要核对真国产 vs 名义国产
