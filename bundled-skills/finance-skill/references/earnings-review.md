# 财报后反应（earnings-review）

> **何时参考**：财报刚发布、业绩会刚结束，用户问"财报怎么样 / 业绩会说了啥 / 为什么超预期但不涨 / 财后该不该买"。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

**同时解释"财报内容"和"市场反应"**——结果好坏是一回事，股价反应强弱是另一回事。

## 分析步骤

1. **确认事实与价格反应窗口**：财报是否完整披露、股价在盘后 / 盘前 / 次日主要反应、反应幅度是否超出历史常态、同板块可比是否联动。价格还在快速变化时声明"判断仅代表当前时点"。
2. **拆开"结果"与"表现"**：
   - 收入 / 利润 / 利润率 / 现金流核心结果是否超预期
   - 经营指标是否支持
   - 指引是否上修 / 持平 / 转弱
   - 增长质量是来自主业 / 结构优化 / 还是一次性
   - 价格反应是放大财报结论还是与财报错位
3. **识别市场真正交易的核心分歧**：增长可持续性 / 利润率触顶或修复 / 某业务线是否进入拐点 / 管理层指引是否反映更深层问题 / 原本定价的故事是否被强化或削弱。**若股价与 headline 相反，重点解释市场为何忽略表面利好。**
4. **业绩会提炼**：管理层措辞变化（去年的"强劲增长"今年变成"稳健增长"）、问答环节的市场分歧定位、新增信息 vs 重复口径、是否有指引上下调。
5. **判断财报后持续性**：反应是一次性情绪释放还是趋势级重估起点、哪些细节会在次日 / 次周继续交易、若市场解读过度后续如何回摆。
6. **更新研究判断**：原有投资逻辑被强化 / 削弱 / 基本未变、是否需要调整优先级、若继续跟踪下一步最值得验证什么。

## 建议骨架（不强制）

- 30 秒结论（结果层 / 反应层 / 被交易的变量 / 当前判断）
- 财报结果拆解表（项目 / 预期 / 实际 / 解读）
- 为什么市场这样反应（正向驱动 / 压制因素 / 表面与实质差异）
- 财报后真正的分歧点
- 后续看什么（短期验证 / 中期验证 / 改变判断的信号）
- 当前动作建议

## 避坑

- 同时解释财报内容 + 市场反应，不能只写一半
- 股价与 headline 不一致时，优先解释错位原因
- 不能把所有变动归因为单一指标，要突出**真正决定反应的主变量**
- 盘后情绪当长期结论是常见错误——带时间窗口意识
- 财后漂移（PEAD）是真实存在的：超预期后的趋势可能延续 1-3 个月，不要把首日反应当全部
