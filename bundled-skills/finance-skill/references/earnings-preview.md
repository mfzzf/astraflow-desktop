# 财报前瞻（earnings-preview）

> **何时参考**：用户在财报发布前问"X 财报会怎么样 / 有什么看点 / 会不会超预期 / 财报前是否买入 / 关注什么数据"。

## 核心目标

**围绕"预期差"展开**——市场已知什么 / 已押注什么 / 还在争论什么。

## 分析步骤

1. **确认对象与目的**：分析对象、财报周期（Q1 / 中报 / Q3 / 年报）、用户是要快速准备、判断参与价值、还是持仓风险评估。
2. **建立市场预期基线**：
   - 收入 / 利润 / 利润率核心指标的市场预期区间（卖方一致预期）
   - 管理层上次指引
   - 卖方讨论最集中的变量
   - 上一季度遗留的验证任务
   - 当前股价隐含的预期温度（保守 / 中性 / 偏热）
3. **本次最关键 3-5 个验证点**：
   - 是否只表面超预期还是质量同步改善（毛利率 / 经营性现金流跟得上吗）
   - 增长来源是量 / 价 / 结构还是一次性
   - 利润率 / 费用率 / 现金流是否支持增长叙事
   - 订单 / 用户 / 产能 / 库存等经营变量是否验证主线
   - 管理层是否需要回应市场最大疑虑
4. **情景推演与反应框架**：拆开"财报结果"和"市场反应"——
   - 明显超预期：股价怎么走
   - 略超但股价不涨：可能是预期已过满
   - 表面达标但指引偏弱：杀估值风险
   - 不及预期：跌幅看哪些条件
5. **会前关注清单**：财报发布后先核对的 3 个数字、业绩会最值得听的问题方向、哪些措辞变化会明显改变判断、哪些风险即使财报不错也不能忽略。

## 建议骨架（不强制）

- 30 秒结论（最重要看点 / 最关注变量 / 最容易出现的预期差 / 整体判断）
- 市场预期基线表
- 本次最关键的验证点（3-5 条）
- 情景推演（超预期 / 平稳分化 / 不及预期）
- 财报前风险提示
- 会前问题清单

## 避坑

- 围绕预期差展开，不要重复公司背景
- 关键看点尽量具体到指标 / 业务模块 / 管理层表态方向
- 区分"财报结果好坏"和"股价反应强弱"
- 财报前**不给点位预测**，只给情景与触发条件
- 美股财报盘前 / 盘后发布要点出本地时间 + 北京时间双标
