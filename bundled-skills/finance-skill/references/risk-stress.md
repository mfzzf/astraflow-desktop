# 风险压力测试（risk-stress）

> **何时参考**：用户问"VaR 怎么算 / 压力测试 / 极端情况下亏多少 / 尾部风险 / 蒙特卡洛模拟"。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

**量化极端情景下的最大损失**——给一组完整的风险画像，不只一个数字。

## 分析步骤

1. **VaR（Value at Risk）**：在 X% 置信度下的最大预期损失。
   - 三种计算方法：历史模拟 / 参数法 / 蒙特卡洛
   - 95% 置信度适合日常风险评估
   - 99% 置信度适合极端风险评估
   - **VaR 只给一个点而非完整风险画像**——配合 CVaR 使用
2. **CVaR / ES（条件 VaR / 期望缺额）更保守**：
   - 衡量超过 VaR 阈值的平均亏损
   - 通常 1.3-1.8 倍于 VaR
   - 满足次可加性，符合风险分解原则
3. **最大回撤分析**：从高点到低点的最深亏损幅度、回撤持续时间、恢复时间——反映历史最坏情景。
4. **蒙特卡洛模拟**：10000 条路径、几何布朗运动假设、产出完整分布；输出概率分布、百分位数、极端概率计算。
5. **压力测试场景**：
   - **历史情景**：2008 金融危机、2015 股灾、2020 疫情、2022 加息周期等
   - **假设情景**：加息 100bp / 信用危机 / 流动性干涸 / 地缘冲突
   - 逐情景计算组合损失与是否触发止损
6. **尾部风险分析（EVT）**：
   - 拟合广义帕累托分布
   - 识别尾部是肥尾（形状 ξ>0，危险）还是薄尾（ξ<0，安全）
   - 中国 A 股常见 4-8 的过度峰度，左偏斜（大跌频繁）
7. **关键指标组合**：波动率、最大回撤、偏度、峰度、VaR（95%/99%）、CVaR、Sharpe 比率、Information Ratio。

## 建议骨架（不强制）

- 核心风险指标汇总（波动率 / 回撤 / VaR / CVaR / 偏峰）
- 多置信度 VaR 对比（95% vs 99%）
- 压力测试结果（历史情景 5-10 个 + 假设情景 3-5 个）
- 蒙特卡洛模拟分布与百分位（最坏 1% 情景）
- 尾部风险判断（肥尾 vs 薄尾）与控制建议

## 避坑

- VaR 不是最大损失——只是置信度内的损失，剩余 5% 可能更糟
- 正态分布假设对金融数据过于乐观——金融回报常见肥尾，参数法 VaR 严重低估极端风险
- 历史模拟法受样本影响大——极端行情外的数据容易被忽视
- 压力测试场景需定期更新——20 年前的危机特征不代表未来
- 相关系数在极端行情时往往急剧上升（从 0.3→0.9）——组合多元化收益大幅折扣
