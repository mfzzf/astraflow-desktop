# 板块比较与轮动（sector-comparison）

> **何时参考**：用户问"X 板块和 Y 板块哪个更值得 / 板块怎么选 / 板块轮动 / 哪个赛道现在好 / 风格切换"。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

**对多个板块做横向比较并给排序**，不是把每个板块单独写一遍。

## 分析步骤

1. **核心逻辑对比**：每个板块当下市场为什么买、未来靠什么继续涨——**不是列公司名单**。
2. **交易位置对比**：低位预热 / 启动确认 / 主升演绎 / 高位拥挤 / 退潮修复——**同样的逻辑在不同位置性价比完全不同**。
3. **催化强度对比**：硬催化（业绩 / 政策 / 订单）vs 软催化（情绪 / 海外映射）；存量叙事 vs 预期透支。区分哪个方向催化更连续、更容易扩散。
4. **资金偏好对比**：成交额 / 龙头强度 / 中军承接 / 机构 + 游资参与度 / 趋势持续性，看哪个板块资金更愿意聚焦。
5. **拥挤度与估值压力**：逻辑好但过度拥挤的板块短期风险反而大——平衡逻辑与位置。
6. **风格轮动判断**（如果涉及大小盘 / 成长价值切换）：流动性环境（宽松利成长 / 紧缩利价值）、盈利预期斜率、政策方向、市场情绪温度。
7. **必须给排序**：短期优先级 / 中期优先级 / 是追踪还是等待。**不给排序就不算"比较"。**

## 建议骨架（不强制）

- 核心逻辑对比
- 交易位置对比（高 / 中 / 低）
- 催化强度与持续性
- 资金关注度与赚钱效应
- 拥挤度评估
- 优先级排序与配置建议

## 避坑

- 不是"逐个分别写"——是"横向比较"
- 不只看逻辑强弱，必须结合位置和交易性
- 高估值不一定不买，关键看相对性价比
- 板块轮动是结果不是预测——别用后视镜推断"下一个会是哪个"
