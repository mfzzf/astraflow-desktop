# 因子研究（quant-factor-research）

> **何时参考**：用户问"动量因子 / 价值因子 / 多因子选股 / IC / IR / 因子有效性 / 怎么验证因子"。

## 核心目标

**系统性检验单因子或多因子的选股能力**——通过 IC / IR 与分层回测判断因子有效性。

## 分析步骤

1. **IC（Information Coefficient）**：因子值与后期超额收益的相关性
   - IC > 0.03：基本预测力
   - IC > 0.05：强预测力
   - IC > 0.1：需检查前向偏差（look-ahead bias）
2. **IR（Information Ratio）= IC 均值 / IC 标准差**
   - IR > 0.5：稳定有效
   - IR > 1.0：极罕见，需复核
   - IC 正负比例 > 55%：方向稳定
   - < 50%：方向不稳定，不可用
3. **分层回测**：
   - 按因子值从低到高分成 N 组（默认 5 组）等权持有
   - 观察最低组 → 最高组的**单调性**（好因子应单调上升 / 下降）
   - 长短空差价（spread）越大越强
   - 中间组平滑程度体现稳定性
4. **因子组合方法**：
   - 等权（最简单）
   - IC 加权（高 IC 因子权重更大）
   - 正交化（Gram-Schmidt 处理消除相关性）
   - 复杂度与增益要平衡
5. **常见陷阱与修正**：
   - **前向偏差**：因子值必须用 T 日及之前数据计算，收益必须是 T+1 至 T+N 的前向收益
   - **分布偏度（市值 / 换手）**：原始值算 IC 会被极值主导——用排序或 Z-score 标准化
   - **行业偏见**：因子值常在同行业相似——做行业中性化（行业内 Z-score）
   - **样本不足**：每截面至少 5 个有效标的
   - **因子拥挤**：经典因子（动量 / 价值）可能因被广泛使用而衰减——定期监测时间序列演化
6. **单因子有效性阈值**：IC > 0.03 且 IR > 0.5 才考虑组合；回测覆盖周期 3-5 年，含完整市场周期。

## 建议骨架（不强制）

- 因子定义与计算方法
- IC / IR 统计汇总（均值 / 标准差 / 正率 / 有效期）
- 分层回测权益曲线与长短差价
- 分时间段 / 分行业 / 分风格的因子效力
- 多因子组合权重与回测对比基准
- 因子失效预警与更新建议

## 避坑

- 高回测 IC 不代表实盘能实现——滑点、成本、择股偏差侵蚀收益
- 样本外测试（walk-forward）必须严格执行
- 因子排名的边际效用递减——顶部 1% 和顶部 10% 选股效果可能差异不大但成本差异大
- 因子组合数量过多容易过拟合——宁可用 5-10 个强因子也不用 30 个弱因子
- 不同持有周期 / 不同市场环境下因子效力差异巨大——分层验证

## 可执行工具（scripts/quant/）

⚠️ **数据 skill 边界**：westock / westock-data 给你**因子原料**（PE / PB / ROE / 收益率序列），但**因子有效性的统计验证（IC / IR / 分层回测 / 前向偏差处理 / 行业中性化）数据 skill 不做**——这些是算法实现，不是数据查询。**做新因子有效性验证必须用 script，不要让 model 凭记忆模拟 IC 计算**。

| 文件 | 用途 | 行数 |
|---|---|---|
| `factor_multi.py` | 多因子选股引擎（IC/IR 计算 + 分层回测 + 因子等权 / IC 加权 / 正交化组合） | 202 |
| `factor_fundamental.py` | 基本面因子筛选（PE / PB / ROE / 增长率多维度打分） | 127 |

**Setup**：`pip install pandas numpy requests`（无特殊依赖）。

**调用示例**：
```python
import sys; sys.path.insert(0, 'scripts/quant')
from factor_multi import SignalEngine
engine = SignalEngine()
signals = engine.generate({"600519": ohlcv_with_factors_df})
```

**做新因子有效性验证时优先用 `factor_multi.py` 现成的 IC/IR 计算逻辑**，不要从零重写 — 它已经处理了前向偏差、行业中性化、Z-score 标准化等坑。
