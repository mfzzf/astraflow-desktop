# 估值与定价（valuation-pricing）

> **何时参考**：用户问"贵不贵 / PE 多少 / 估值 / 是否高估 / 重估空间 / 杀估值风险 / DCF / PEG / PB / 同行估值对比"。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

回答三件事：**该用什么估值方法、当前估值反映了市场什么预期、未来重估或杀估值的触发因素**。

> **取数提示**：算历史估值分位、估值中枢要长周期——westock-data 等官方源默认只返回约 3 年，看不出中枢和分位。取数主动写"近 10 年"（如"X 近 10 年 PE PB 历史分位"），3 年根本算不准"历史 70% 分位"这类判断。

## 分析步骤

1. **判断公司类型**：高成长 / 稳健成长 / 周期 / 价值 / 资源 / 平台型 / 金融——不同类型不能用同一套方法。
2. **选择估值框架并解释为什么**：
   - 高成长：PE / PEG / PS / EV-EBITDA
   - 稳健成长：PE / PEG
   - 周期：PB / 周期中枢 PE / 盈利拐点
   - 价值：PE / PB / 股息率
   - 平台 / 金融：分部估值 / DCF / P/AUM 等
3. **拆解当前估值反映的预期**：高估值隐含未来高增速 / 景气持续 / 市占率提升 / 新业务高弹性；低估值隐含周期下行担忧 / 业绩见顶 / 治理折价 / 兑现能力不足。**说清"股价里已经包含了什么"**，是高质量估值分析的关键。
4. **估值位置判断**：3 年历史分位（30% 以下偏低、30%-70% 中性、70% 以上偏高），同时行业横向对比，结合公司质地与阶段——高估值不一定贵、低估值不一定便宜。
5. **重估 / 杀估值触发因素**：
   - 重估来自：业绩超预期 / 产业趋势强化 / 商业模式改善 / 资金风格切换 / 治理改善
   - 杀估值来自：增速下修 / 景气拐点 / 竞争恶化 / 风险偏好下降 / 政策冲击
6. **DCF 关键参数（如果用）**：WACC 区间通常 8%-12%，永续增长率 2%-3%，明确做 Bear / Base / Bull 三情景，**不要把单一情景当目标价**。
7. **形成投资判断**：当前估值是否有性价比、投资重点是赚业绩钱还是赚估值钱、当前更适合乐观 / 审慎 / 等待。

## 建议骨架（不强制）

- 公司类型判断
- 选择的估值框架与理由
- 当前估值所反映的预期（拆解市场已定价了什么）
- 估值位置（历史纵向分位 + 行业横向）
- 重估 / 杀估值触发因素
- 投资判断（赚业绩钱 vs 赚估值钱）

## 避坑

- 不生搬硬套——必须解释为什么这个方法最适合这家公司
- 不能只给 PE 数字就下结论，必须拆解估值背后的预期
- 高估值不是"一定要卖"信号，看增速与质量是否匹配
- 跨公司比 PE 前先核对财年口径（财年 vs 自然年）
- 不给具体目标价点位预测，可给估值区间和情景

## 可执行工具

⚠️ **数据 skill 边界**：westock 给的是 PE / PB / 历史估值分位等**已计算指标**，DCF 模型本身和 Excel 校验都不在数据 skill 范围内。

用户提供 DCF Excel 模型让你检查时，先跑 `scripts/ib/validate_dcf.py`：

```bash
python3 scripts/ib/validate_dcf.py <path/to/dcf_model.xlsx>
```

它会自动检查：公式错误（#REF! / 循环引用）、WACC 合理性、永续增长率边界（建议 < GDP 增速）、terminal value 占比（>70% 警示）、敏感性分析完整性等。**比手工检查 Excel 快且不漏项**。

**Setup**：`pip install openpyxl`（未装报 `ModuleNotFoundError: No module named 'openpyxl'`）。
