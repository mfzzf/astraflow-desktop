# 每日投研简报（daily-briefing）

> **何时参考**：用户问"今天市场怎么样 / 盘前简报 / 收盘复盘 / 晨会纪要 / 自选股新闻 / 每日观察 / 盘后总结"。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

**生成日度市场总结 / 盘前简报 / 收盘复盘**——结构标准化，内容有判断密度。**不做数据搬运工**：每个区块都要有"所以呢"——数据背后的含义、对用户持仓 / 关注领域的影响、可操作的判断；纯数据罗列是失败的输出。

## 盘前 / 盘后双模式（按触发时点自动切换侧重点）

两种模式共享同一分析框架，但侧重不同：

| 模式 | 触发时点 | 侧重内容 |
|---|---|---|
| **盘前简报**（如交易日 08:30）| 美股已收盘、A/HK 即将开盘 | 隔夜全球要闻、隔夜行情（美股 / 大宗 / 汇率）、对今日 A/HK 的影响预判、自选股盘前扫描 + 竞价前瞻、今日关注什么 |
| **盘后简报**（如交易日 16:30）| A/HK 已收盘、美股即将 / 刚开盘 | 收盘总结（指数 / 涨跌家数 / 成交）、板块涨跌排行、资金流向、自选股当日复盘、明日展望 |

**结构标准化区块**（按模式取舍）：盘前=隔夜要闻 / 全球行情 / 指数预判 / 自选股盘前扫描 / 今日关注；盘后=收盘总结 / 板块涨跌 / 资金流向 / 自选股复盘 / 明日展望。

## 资讯搜索是核心数据源，不可跳过

- 盘前必做"全球隔夜资讯搜索"、盘后必做"当日市场资讯搜索"，**至少覆盖 3 个主题**（宏观 / 地缘 / 大宗 / 北向 / 板块热点等）
- 资讯直接决定「隔夜要闻 / 今日市场 / 板块表现 / 明日展望」区块的质量——**没搜索 = 报告严重缺失**，比行情数据缺失更不可接受
- 别因为"已经有行情数据了 / 内容够了"就主动跳过搜索；只有搜索工具全不可用时才降级标注
- 数据源优先 westock-data / westock-data，资讯类用 WebSearch 兜底（标来源 + 日期）

## 数据职责防错（真实事故，必须遵守）

- **个股涨跌幅只能来自行情数据（westock-data quote / westock-data），不能从"自选股列表"里读**——自选股工具的职责仅是给出"用户关注哪些代码"，其附带的百分比字段含义可能是"日内振幅"而非"涨跌幅"
- 已知事故：误用自选股附带字段，把腾讯实际 -3.56% 报成 -4.75%、阿里 -4.47% 报成 -5.38%
- **正确流程**：① 拿自选股代码列表 → ② 用行情工具批量查这些代码 → ③ 从行情结果读涨跌幅。所有市场（A / 港 / 美）一视同仁，不能港美股图省事直接用列表字段

## 输出纪律（简报型推送的硬要求）

- **直接给报告本身，不要过渡语**：禁止"当前时间是…""数据已到位""以下是分析结果""让我整理一下"等任何前言 / 中间推理 / 状态播报
- **标题是第一行**，标题前不放分隔线（推送弹窗只显示前 1-2 行，第一行要能一眼识别来源）；标题后不附加说明文字（非交易日就用最近一个交易日日期）
- **红涨绿跌**（中国市场惯例，与西方相反）—— 面向 A/H 用户必须遵守
- **异动阈值**（筛选要展示的自选股）：涨跌幅 5% / 成交量 2 倍为默认，用户可调
- **只展示有实质变化的维度**：某只自选股没有价格 / 消息 / 资金变化的维度直接省略，不要写"无明显变化""暂无消息"之类的占位内容
- HTML 落盘呈现见文末"输出形式"；若是定时推送场景，则按上述纪律输出精简 dashboard

## 简报结构（标准化）

1. **市场概览**（指数走势 / 成交量 / 涨跌家数 / 行业表现）
2. **新闻导读**（重点事件 / 政策发布 / 上市公司公告 + 简评）
3. **盘中交易热点**（龙虎榜 / 资金流向 / 板块轮动）
4. **个股精选**（自选股表现 / 重点跟踪变化）
5. **技术面观察**（关键支撑阻力 / 图形破位 / 均线位置）
6. **后市研判与策略**（短期风险 / 配置建议）

每个 section 控制在 150-300 字，言简意赅。

## 内容深度的四层递进

1. **事实层**：发生了什么（政策发布、数据发布、个股公告）
2. **解读层**：为什么重要（对哪些行业 / 个股有影响）
3. **映射层**：股价含义（预期涨跌逻辑）
4. **操作层**：怎么做（增仓 / 减仓 / 观望）

## 资金流向解读

- 北向资金净流入 / 流出（外资对 A 股的风险偏好）
- 两融余额（融资杠杆风险）
- 龙虎榜（识别游资集团与机构追捧方向）
- 产业资本增减持（insider signal，常先于股价 1-2 周反应）

## 技术面信号优先级

1. 趋势（日线上升 vs 下降）
2. 位置（距离关键支撑 / 阻力多远）
3. 形态（头肩顶 / 底、三角形突破、圆底）
4. 量能（volume confirmation）

**千万不要反向操作**（趋势向下还做多 / 趋势向上还看空）。

## 板块轮动维度

- 周期板块：受宏观景气驱动，顺周期
- 防御板块：逆周期，下行期相对稳定
- 成长板块：自身基本面与政策扶持驱动
- 根据宏观环境与情绪判断当期应该倾斜配置哪个

## 策略建议表述框架

- **短期 T+1 至 T+5**：focus on technical & sentiment，说清 nearest resistance / support、量能是否跟随
- **中期 1-3 个月**：focus on fundamental catalysts，说清接下来哪些数据 / 公告会验证 thesis
- **长期 3-12 个月**：focus on business fundamentals & valuation，说清 growth story 是否 valid、当前估值是否 reasonable

## 常见误导信号与避坑

- 单日跌幅大 ≠ 趋势反转（多日确认）
- 龙虎榜游资追捧 ≠ 值得跟（往往是炒作尾声）
- 北向大额净流出 ≠ 必然下跌（可能是风险分散 rebalance）
- 利好消息发布 ≠ 股价必涨（"buy the rumor, sell the news"）

## 建议骨架（不强制）

- 市场概览（指数 / 成交 / 涨跌家数 / 行业涨跌排行）
- 新闻导读（3-5 条重点事件 + 简评）
- 盘中热点（龙虎榜 / 资金流 / 板块轮动）
- 自选股追踪
- 技术面观察（支撑阻力 / 重要形态）
- 后市研判与建议

## 避坑

- 不要"每日预测"涨跌方向——错误率 > 50%，改为"观察重点 + 风险 / 机会框架"
- 不过度解读单日数据——市场波动正常，多周期确认趋势
- 不被市场 noise 迷惑（某游资集团追捧）——必须问"逻辑是否可持续"
- 不推荐超出认知范围的标的——承认"暂无 clear signal"比错误推荐更专业
- 不用绝对语言（"必涨""必跌"）——用概率表述
- 美股 / 港股事件时点要点出本地时间 + 北京时间双标
- 个股涨跌幅别从自选股列表字段读（可能是振幅）——必须走行情数据
- 别写"无变化"占位——没变化的维度直接省略
- **每日简报 ≠ 实时盯盘**：用户说"涨跌超 X% 提醒我"在简报语境里是调"异动展示阈值"；只有明确"实时监控 / 盯盘 / 立刻通知"才是 monitor-alert / 实时监控工具的场景，别混
