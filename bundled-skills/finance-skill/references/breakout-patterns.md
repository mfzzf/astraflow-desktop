# 波缩突破与 VCP（breakout-patterns）

> **何时参考**：用户问"突破怎么看 / 这个形态像不像突破 / VCP / 波动收缩 / 假突破怎么识别 / 突破后能不能追"。

## 核心目标

**识别波动逐级收缩、供给减弱、结构成熟的突破预备股**——并区分真突破和假突破。

## 分析步骤

1. **VCP 结构核心是波动 + 供给收敛**（不是单纯横盘）：
   - 回调幅度逐步收窄（每轮回调比上轮浅）
   - 量能同步收敛（回调阶段量能逐步萎缩）
   - 不是一次大跌后静止
2. **前置强势基础**：进入收缩期前是否已有清晰上涨段、前期上涨是否有基本面 / 题材 / 资金支持。**没有强势基础的横盘难以构成高质量 VCP**。
3. **波动逐级收缩判断**：
   - 每轮回调幅度比上轮更浅
   - 修复后的高点更接近前高
   - 日内波动呈压缩趋势
   - 后期明显减少剧烈震荡
   体现"卖压在减少"而非"交投停滞"。
4. **量能同步收敛**：
   - 回调阶段量能逐步萎缩
   - 反弹阶段偶有适度放量显示承接
   - 临近关键位时量能温和回升
   - 不存在某次放量下跌破坏供需平衡
5. **关键位置与触发条件**：突破触发位、最关键供需转折点、突破时所需量价确认（通常需要量能放大 1.5-2 倍 + 收盘站稳）、失败判断边界。
6. **基本面 / 催化支撑**：技术结构漂亮 ≠ 高质量——需确认业绩 / 产品 / 题材 / 订单或行业景气支持。
7. **成熟度分层**：结构成熟接近触发 / 收缩正在形成 / 结构待修复——分别对应不同的关注度。

## 建议骨架（不强制）

- 候选股 VCP 成熟度排序
- 候选池总览（前置强势 / 波动收缩 / 量能收敛 / 触发准备度评分）
- 重点候选拆解（结构特征 / 关键位置 / 理想确认信号 / 欠缺条件 / 失效边界）
- 观察清单与验证指标
- 风险提示（假突破识别）

## 避坑

- 容易把普通横盘 / 弱势震荡误判为 VCP
- 成交量极低的"整理"可能是无人接盘而非供给减弱——区分
- 技术图形漂亮但基本面转差的突破往往是假突破
- VCP 形成周期通常长于预期，等待过久会错失别的机会
- 突破后缩量回踩是健康的——放量回踩是失败信号

## 可执行工具（scripts/price-action/）

⚠️ **数据 skill 不覆盖 VCP / 波缩突破形态检测**——这种形态本质是"识别波动逐级收窄 + 量能同步收敛"的结构特征，westock-data 和 westock-data 都只能给你 OHLCV 原始数据，**形态判断完全靠你自己写算法**。

VCP 检测现在有统一 CLI，优先用一条命令完成 K 线拉取 + 波动收缩 / 量能收缩 / 触发价 / 失效位评分：

```bash
python3 scripts/run_signal.py --engine vcp --source westock --code sz300308 --limit 120 --pretty
```

| 文件 / 命令 | 在 VCP 检测里的作用 |
|---|---|
| `scripts/run_signal.py --engine vcp` | **首选入口**：输出 score、status、trigger_price、invalid_below、4 段收缩结构 |
| `scripts/price-action/basic_indicators.py` | 底层指标：波动率收敛 + 量能收敛 |
| `scripts/price-action/candlestick_patterns.py` | 底层确认：突破触发日 K 线形态确认（吞噬 / 三白兵等强势信号） |

**典型组合流程**：
1. 用 `run_signal.py --engine vcp --source westock --code <code>` 得到结构评分
2. 若 `status=mature_watchlist/形成中`，再结合主升前置阶段、基本面催化、关键位确认做最终判定

**Setup**：`pip install pandas numpy requests`（无特殊依赖）。

调用约定见 `price-action-tools.md` 末尾的"可执行工具"section。
