# 公告与股东信（announcement-impact）

> **何时参考**：用户问"X 出了 Y 公告怎么看 / 收购了 Z 怎么看 / 减持影响 / 定增 / 重大合同 / 拆分 / 股东信解读"。

## 核心目标

**先讲事实再讲影响**，把直接影响和二阶影响分层，不混成泛论。

## 分析步骤

1. **确认公告类型与核心事实**：并购 / 出售 / 分拆 / 重组、减持 / 增持 / 定增 / 配股、重大合同 / 订单 / 诉讼、产能建设 / 资产处置 / 控制权变化。梳理：谁参与、金额多大、时间安排、条件限制、是否需审批。
2. **建立事件前基准状态**：公司原有经营和估值状态、市场原本最担心或最期待的、事件是否早有传闻或已部分定价、事件是补强主线还是改变主线。
3. **拆解直接与二阶影响**：
   - **直接**：收入 / 利润 / 现金流 / 资产负债表
   - **二阶**：控制权 / 治理 / 融资能力 / 股东回报结构性影响
   - **中期**：行业地位 / 竞争格局 / 战略方向
   - **短期**：市场情绪和估值
   - 区分立即可见与后续传导
4. **识别谁受益 / 受损 / 观望**：公司本身、原股东、新投资方、管理层、债权人、供应链等利益相关方；是否存在外溢到同行 / 上下游 / 题材板块。
5. **评估不确定性与落地门槛**：审批 / 交割 / 执行节奏风险、价格 / 估值 / 对价结构争议、市场是否高估短期收益、低估长期成本、哪些条件未满足需推翻解读。
6. **股东信特有**：提炼信件最频繁关键主题（战略方向 / 竞争优势 / 客户价值 / 产品路线 / 资源投入）；对比"哪些没变、哪些变了"；判断是否有经营结果支撑、执行抓手是否明确、可信度如何。

## 建议骨架（不强制）

- 30 秒结论（性质 / 核心影响 / 最先受影响的层面 / 最大不确定性）
- 事实概览表
- 影响链条拆解（直接 / 二阶 / 被忽视）
- 受益与受损方
- 落地风险与后续观察
- 当前动作建议（优先跟踪 / 谨慎乐观 / 保留 / 等待落地）

## 避坑

- 先讲事实再讲影响，不先入为主
- 直接与二阶影响分层，不混成泛论
- 不把"讲故事空间"误写成"已兑现价值"
- 事件需审批 / 交割时，把风险放在核心位置
- 减持 / 定增不要机械等同利空——要看动因和执行条款
