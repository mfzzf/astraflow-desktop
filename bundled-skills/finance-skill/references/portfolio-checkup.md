# 持仓体检与风控（portfolio-checkup）

> **何时参考**：用户给出多只持仓（或自选股），问"我的组合怎么样 / 持仓体检 / 风控分析 / 该砍哪只 / 解套思路"。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

**从组合视角思考，不从逐只点评切入**——先讲清组合画像、再点结构性问题、最后给具体调整建议。

## 分析步骤

1. **组合总体画像**：行业分布、风格分布（成长 / 价值 / 周期 / 防守）、持仓数量、集中度、总仓位水平。**让用户先知道自己拿的是什么。**
2. **主要风险暴露识别**：单一行业过重、同类题材高度集中、高波动个股暴露、高相关性标的重复（3 只半导体 = 1 笔半导体仓）、风格冲突（既想防守又高弹性）。
3. **结构性问题点名**：
   - 龙头缺失，后排太多
   - 逻辑混乱，无统一框架
   - 跟风股过多
   - 票数过多导致跟踪失效（>15 只对个人投资者基本失控）
   - 过度分散拖累弹性 / 过度集中放大回撤
4. **持仓分层**：
   - **进攻核心仓**：逻辑强 + 趋势好 + 风险收益比佳
   - **稳定器仓**：低波动、长期持有
   - **观察仓**：逻辑待验证 / 仓位轻
   - **问题仓 / 拖累项**：逻辑弱 / 走势差 / 占用资金成本
5. **总仓位判断**：偏高 / 合理 / 偏低，结合市场状态给出"该不该减总仓"的判断。
6. **调整优先级（明确顺序）**：
   - 先砍逻辑最弱的跟风仓
   - 再优化重复暴露
   - 再提高核心仓位
   - 最后考虑加新标的
7. **解套场景**（持仓亏损时）：分清"是否值得继续持有"——逻辑还在 + 价格被错杀就补仓，逻辑破坏就割肉换股，**不要简单按浮亏比例决定**。

## 建议骨架（不强制）

- 组合总体画像
- 主要风险暴露
- 结构性问题点名
- 核心仓 / 观察仓 / 问题仓分类
- 仓位与风格调整建议
- 调整执行优先级

## 避坑

- 不能从逐只股票角度简单点评，必须从组合视角
- 不要空喊"风控"，要指出具体风险暴露和量化集中度
- 允许明确批评式结论，不要假大空
- 解套不是机械补仓，要重新审视逻辑
- 票数太多本身就是风控问题
