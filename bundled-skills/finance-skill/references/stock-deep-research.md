# 个股深度研究（stock-deep-research）

> **何时参考**：用户要求对一只股票做卖方研究员级别的深度分析（"深度分析"、"完整研究"、"投资逻辑"、"研究备忘"），覆盖商业模式、竞争格局、投资逻辑、成长驱动、估值与风险等多维度。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

不是资料堆砌，而是回答两件事：**市场为什么会买这只股票，未来还会不会继续买**。

## 数据时效与锚定（动手前先做）

- **按当前日期推算"理论上应已披露的最新财报期"**：如当前 2025-05，A 股应已有 2024 年报 + 2025 一季报；用来判断拿到的数据是不是最新一期
- **取历史财务默认要长周期**：westock-data 等官方源**不指定年限时默认只返回约 3 年**，看不出周期和估值中枢。深度研究 / 估值分位 / 周期判断时，query 里**主动写"近 10 年"**（如"X 近 10 年营收净利 ROE 毛利率"），实测能拉满 10 年年报，且上市不足 10 年的次新股会自动按实际上市年限返回、不报错
- **官方源自动返回最新期 + 自带时点**：直接问即可，westock-data 会一次返回最新季报 + 多期年报且每条带披露日 / 统计截止日，**不需要像 web 搜索那样在 query 里硬塞时间关键词来防旧数据**
- **核心数据（收盘价 / 市值 / 估值）跨源出现分歧时**，注明差异和采用口径（如 PE TTM vs 静态 PE）
- 只有降级到 WebSearch 兜底时，才需要在搜索词里带时间关键词（`X 2025 一季报` 而非 `X 财报`）避免返回旧文章

## 分析步骤

1. **拆解公司与业务结构**：哪块业务最关键、哪块决定估值、是否有真壁垒。区分主业务、增长业务、衰退/边缘业务。
2. **行业格局与公司位置**：行业阶段（导入 / 成长 / 成熟 / 下行）、行业空间、竞争强度、公司位置（龙头 / 细分龙头 / 追赶者 / 边缘）。
3. **核心投资逻辑（2-4 条）**：每条必须包含——逻辑本身、为什么成立、对业绩或估值的影响路径、是否已被充分定价。**不接受**"受益于行业发展"这种泛逻辑。
4. **未来 1-3 年增长驱动**：来自产品放量、价格上行、市场份额、新业务、成本改善哪一个；明确**最关键的业绩变量**。
5. **市场共识与预期差**：市场已普遍接受什么、真正可能形成超额收益的预期差是什么（被低估的盈利弹性 / 被高估的短期风险 / 被忽视的新业务 / 周期判断滞后）。这是高质量投研的关键。
6. **估值方法选择与位置**：PE / PEG / PB / EV-EBITDA / 分部估值——为什么选这个方法，当前处于历史 3 年分位什么位置，行业横向对比如何。
7. **真正可能伤害股价的因素**：短期与中期风险分开（需求 / 价格 / 客户集中 / 产能 / 政策 / 竞争），不要泛泛提示。

## 建议骨架（不强制）

- 公司核心信息与业务结构
- 商业模式与竞争力拆解
- 行业格局与公司地位
- 核心投资逻辑（2-4 条）+ 关键变量
- **一句话本质**："买这家公司本质上是在买 ____"（如"买的是 AI 算力资本开支周期的量价齐升""买的是管理层困境反转的执行力""买的是锂价见底后的周期弹性"）——逼自己把投资逻辑收敛成一句可证伪的话
- 市场共识与预期差
- 估值框架与当前位置
- 风险（短期 / 中期分开）
- 投资结论（短期视角 + 中期视角）

## 避坑

- 不写公司介绍式资料堆砌
- 每条投资逻辑要具体到影响路径，不接受"受益于行业景气"这种泛话
- 预期差是核心交付，不能敷衍
- 数据用来支持判断，不要堆数据没结论
