# 同业比选（peer-comparison）

> **何时参考**：用户问"X 和 Y 哪个更值得买 / 三家比一比 / 同行对标 / 选龙头"，需要在同行业 2-5 家公司中做横向比较并给排序。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

**输出可执行的相对排序与选择理由**，而不是几篇公司介绍拼接。

## 分析步骤

1. **确认可比范围**：所处赛道是否一致、商业模式是否相近、发展阶段是否相似、用户想比短期交易还是中长期配置；不可比的先说明边界。
2. **建立统一比较框架**：业务质量与竞争位置 / 成长速度与确定性 / 盈利能力与现金创造 / 估值与预期 / 近期催化与风险暴露。**所有候选用同一框架**——不能某家只写故事另一家只看数字。
3. **基础数据对齐**：时间口径一致（按自然年季度对齐，见下方）、币种一致、估值方法一致、异常值做说明。
4. **比较"业务质量"而不仅是"数据高低"**：核心产品与客户结构、市场份额与竞争优势、业务模式抗波动能力、管理执行与资本配置风格。短期数据亮眼但稳定性弱要明确指出。
5. **比较"赔率"与"催化顺序"**：谁好但已被充分定价、谁一般但预期差更大、谁短期有催化、谁需要时间兑现。
6. **输出排序与决策建议**：排名靠前的核心理由、排名靠后的主要短板。如果只能研究 / 持有 / 短期交易一只，分别如何选。

## 跨财年同期对齐（关键，常被坑）

**FY 标号不能直接对**——FY 是公司财年壳子，不同公司财年定义不同，同名 FY 季度对应不同的自然年时段。

**正确做法**：把每家最新 FY 拆成它实际覆盖的自然年季度，再按"自然年同季度"配对，或用 TTM 滚动 4 季消除 FY 差异。

### 常见公司财年映射

| 公司 | FY 起止 | 示例（FY26 含什么自然年季度） |
|---|---|---|
| 腾讯 0700.HK | 自然年 | FY26 = 2026Q1 + Q2 + Q3 + Q4 |
| 阿里 9988.HK / BABA | 4 月制（FY26 = 2025-04 至 2026-03） | FY26 = 2025Q2 + Q3 + Q4 + 2026Q1 |
| 苹果 AAPL | 9 月底制（FY26 ≈ 2025-09 末 至 2026-09 末） | FY26 ≈ 2025Q4 + 2026Q1 + Q2 + Q3 |
| 微软 MSFT | 7 月制（FY26 = 2025-07 至 2026-06） | FY26 = 2025Q3 + Q4 + 2026Q1 + Q2 |

### 季度对季度比

按当前时间点（如 2026-06）做腾讯 vs 阿里同期对比，应展开到自然年季度后配对：

| 自然年季度 | 腾讯（FY=自然年） | 阿里（4 月制） |
|---|---|---|
| 2025Q2 | FY25Q2 | FY26Q1 |
| 2025Q3 | FY25Q3 | FY26Q2 |
| 2025Q4 | FY25Q4 | FY26Q3 |
| 2026Q1 | FY26Q1 | FY26Q4 |

**不要**写"腾讯 FY25 vs 阿里 FY26"——这两个 FY 覆盖的自然年时段错位，结论会失真。

### 年度对年度比（推荐 TTM）

**TTM = 最近 4 个完整披露季度求和**——本身就是按自然年季度滚动，自动消除 FY 定义差异，跨市场跨财年的可比性最高。市场默认估值口径（PE TTM）也是这个。

例：截至 2026-06，腾讯和阿里都已披露到 2026 年自然 Q1：
- 腾讯 TTM = FY25Q2 + Q3 + Q4 + FY26Q1（即自然年 2025Q2–2026Q1）
- 阿里 TTM = FY26 全年（也是 2025Q2–2026Q1）
- 两边覆盖的自然年时段一致，可比

### 不可比情形

- 一方季报缺失或还没披露到对方的最新季度——等齐了再比，或退到双方都有的最近一期
- 财年中途变更过定义的——往前回看 8 个季度核对覆盖区间
- 一次性事项扭曲单季度（重组、税收返还、大额减值）——剔除或单独标注

## 建议骨架（不强制）

- 结论摘要（综合首选 / 预期差选择 / 观察对象）
- 统一比较表（公司 / 业务定位 / 增长 / 盈利 / 估值 / 催化 / 风险）
- 相对强弱分析（业务质量 / 成长质量 / 估值赔率 / 催化时点）
- 最终排序与理由
- 决策建议（针对不同投资目的）

## 避坑

- 只比真正可比的对象，必要时先写清不可比之处
- 同一口径——避免各说各话
- 输出重点是"相对强弱与排序"，不是介绍拼接
- 同时比较质量、估值、时点，不能只用单一指标排名
- 跨市场比较（A 股 vs 港股 vs 美股）先核对货币 + 财年口径
