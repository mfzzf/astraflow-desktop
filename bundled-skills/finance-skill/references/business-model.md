# 业务模式拆解（business-model）

> **何时参考**：用户问"这家公司怎么赚钱 / 业务怎么拆 / 商业模式 / 收入结构 / 增长引擎在哪"，需要把公司业务运转方式快速讲清楚。
>
> **输出形式**：本场景结论以 HTML 文件呈现（用 `Write` 工具落盘到当前工作目录，文件名形如 `<topic>-<YYYYMMDD>.html`），对话正文里附 200-300 字摘要 + 文件路径让用户自行打开。

## 核心目标

把公司**怎么赚钱**讲明白——为谁提供什么价值、靠什么持续赚到、增长往哪扩、最容易在哪一环失守。

## 分析步骤

1. **核心问题**：公司解决的是什么问题，为谁提供什么价值，为什么客户愿意付费。一句话讲清。
2. **收入引擎拆分**：按产品线 / 客户类型 / 地区 / 收费模式（一次性 vs 持续 / 订阅 / 服务费 / 销售）拆分；点出**哪块是现金牛、哪块是增长引擎、哪块是边缘业务**。
3. **利润形成逻辑**：主要成本和费用项 → 哪些成本随规模摊薄 → 利润率改善依赖什么 → 是否有高资本开支 / 库存 / 应收压力。
4. **增长引擎与扩张路径（3-5 条）**：客户数 / 客单价 / 渗透率 / 新品类 / 新地区 / 并购整合 / 渠道扩张；同时指出增长**天然上限**与扩张**门槛**。
5. **从反面识别脆弱点（3-5 条）**：单一客户 / 渠道 / 产品依赖、政策 / 原材料 / 补贴 / 景气依赖、竞争加剧时最容易受损的环节、哪个变量最容易让利润模型失真。
6. **下一步最该跟踪的变量**：1-2 个对模型最敏感的变量 + 验证方式 + 失效信号。

## 建议骨架（不强制）

- 一句话讲清公司怎么赚钱
- 业务模块拆分表（模块 / 客户 / 收入来源 / 价值贡献 / 当前角色）
- 利润形成逻辑（盈利来源 / 成本费用关键项 / 利润率敏感变量）
- 增长引擎（3-5 条）
- 业务模式脆弱点（3-5 条）
- 最该跟踪的变量

## 避坑

- 用通俗语言讲，不堆术语和历史沿革
- 收入引擎、利润机制、扩张路径是重点，不泛泛介绍行业
- 优势和脆弱点同时讲清，不单边包装
- 一次性收入和持续性收入要分开标注
