# 监控告警与停复牌（monitor-alert）

> **何时参考**：用户要"设个预警 / 跟踪关键位 / 监控停复牌 / 分红除权前后怎么看 / 目标价到了通知我"。

## 核心目标

围绕**关键位预警 / 目标价触发 / 停复牌跟踪 / 分红变动**，给出明确的位置 + 确认标准 + 触发后的动作。

## 分析步骤

1. **关键位预警**：
   - 位置：前高 / 前低、平台上下沿、中长期均线（60 / 120 / 250）、趋势线、用户预设止损 / 止盈位
   - 重要性说明：为什么这个位置关键
   - 确认标准：向上看突破要量能放大 + 收盘确认 / 次日延续；向下看破位要量能放大 + 跌幅 + 不能快速收回
   - 失真信号：盘中插针、节假日前后异常、消息冲击下的瞬时波动
2. **目标价触发处理**：
   - 到价不代表要卖——判断预期兑现度、是否有新增催化
   - 强势穿越目标 → 重新评估上限
   - 触价即回落 → 重点提示兑现风险
   - 部分止盈 + 留底仓：先卖 1/3 锁利，剩余继续跟趋势
3. **停复牌跟踪**：
   - 停牌原因：重组 / 公告 / 监管 / 财务风险 → 性质截然不同
   - 事件时间线：停牌起止、信息披露节奏、复牌前的预期管理
   - 复牌后流动性与情绪冲击：长停牌后通常有补跌 / 补涨缺口
4. **分红 / 除权变动**：
   - 不机械等同利好 / 利空——要看持续性与支付能力
   - 增加分红：可能是现金流改善 / 看好前景 / 缺乏成长机会**三种**信号
   - 削减分红：经营压力 / 融资需求 / 财务谨慎
   - 特别分红：一次性事项（资产处置）≠ 持续性
   - 除权除息日要点出股价跳空非交易性下跌
5. **告警发送频率与节奏**：盘中级（重要价位）、日级（趋势变化）、事件级（公告 / 停牌）——不同级别用不同通道，避免噪音淹没核心信号。

## 建议骨架（不强制）

- 关键位置与确认标准卡
- 目标价触发处理框架
- 停复牌事件时间线与影响
- 分红变动与持续性评估
- 告警分级建议

## 避坑

- 关键位触碰 ≠ 突破，要收盘确认或次日延续
- 到价处理不机械，考虑预期兑现度与新催化
- 分红变化背后的意图（现金流质量 / 成长见顶 / 资本事项）才是重点
- 除权除息日的"跌幅"不是真跌——别误判趋势
- 港股 / 美股事件时点要点出本地时间 + 北京时间双标

## 实时监控的 14 类预警规则与落地阈值

当用户要"持续盯盘 / 实时监控 / 到价提醒"时，除了上面的方法论判断，还要能给出**可直接配置的预警规则与默认阈值**。下面是经过产品验证的口径（可作为默认值，用户可自定义）：

| 预警类型 | 触发口径 / 默认阈值 | 说明 |
|---|---|---|
| 日内涨跌幅异动 | A股 ±4% / ETF ±2% / 港股 ±3% | 最常用，默认开启；其余规则按需开启 |
| 成本止盈 | 现价相对成本价涨幅达阈值（如 +15%）| 需用户提供成本价 |
| 成本止损 | 现价相对成本价跌幅达阈值（如 -8%）| 需成本价 |
| 动态止盈（追踪止损）| 两段：盈利达 pct1（默认 +5%）启动追踪，回撤 pct2（默认 10%）触发 | 盈利后锁利，需成本价 |
| 成交量异动（放量）| 成交量 ≥ 均量 × 2.0 倍 | 缩量同理可设 |
| 主力资金流 | 净流入 / 流出 ≥ 阈值（如 5000 万 / 1 亿）| A股 / 港股均支持 |
| 均线金叉 / 死叉 | MA5 上穿 / 下穿 MA10 | K 线指标 |
| RSI 超买 / 超卖 | RSI > 70 超买 / < 30 超卖（阈值可调 75/25）| K 线指标 |
| MACD 金叉 / 死叉 | DIF 上穿 / 下穿 DEA | 趋势动能 |
| KDJ 超买 / 超卖 | 比 RSI 更敏感，适合短线 | K 线指标 |
| 布林带突破 | 价格突破通道上 / 下轨 | K 线指标 |
| 跳空缺口 | 开盘价相对昨收跳空超阈值 | 缺口提醒 |
| 公告监控 | 个股有新交易所公告时独立推送（年报 / 业绩 / 减持等）| 噪音低，推荐开启，不依赖价格异动 |
| 每日复盘 | 交易日收盘后（如 15:05）汇总当日涨跌排行 + 当日触发的预警 | 日级汇总 |

**落地注意（性能 / 节奏）**：
- **均线 / RSI / MACD / KDJ / 布林带这类 K 线指标需逐只串行拉历史 K 线**（每只约 35 条），**同时开启这类指标的标的建议 ≤ 20 只**，否则单轮扫描会变慢、影响预警准时性。价格 / 涨跌幅 / 资金流是批量查询，无此限制。
- **扫描频率分时段**：盘中高频（如 5 分钟）/ 午休降频（如 10 分钟）/ 收盘后低频（如 30 分钟），避免无效轮询。
- **告警分级**：盘中级（价位异动）/ 日级（趋势 + 复盘）/ 事件级（公告 / 停牌）分通道，避免噪音淹没核心信号。
- **代码必须查证**：不能凭记忆猜代码（"江西铜业是 sh600362"），必须用数据源 search 确认，保证准确性。

**与本 skill 的分工**：本 reference 讲的是"预警规则怎么定、阈值取多少、触发后怎么判断"；若用户要真正搭一个**后台常驻、零 LLM 的实时监控进程**（daemon + 定时扫描 + 推送到 App），那是工程产品能力，应交给专门的实时监控工具实现，本 reference 提供其规则与阈值的方法论依据。
